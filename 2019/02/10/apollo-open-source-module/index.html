第一课 Apollo开源核心模块概述

Apollo平台技术架构

   百度Apollo的技术架构包括四层：线控车辆平台(Reference Vehicle Platform)、参考硬件平台(Reference Hardware Platform)、软件开发平台(Open software Platform)、云端服务平台(Cloud Service Platform)。
   最底层是线控车辆平台，所有硬件安装在一个线控车辆上面。在线控系统中，驾驶员操作和车辆行为是独立设计的，即输入和输出之间没有机械连接，通过电子信号向车辆施加操作（例如电子油门），这样就可以用电脑来控制车辆，是实现辅助驾驶的基础。
   第二层是参考硬件平台，包括传感器融合单元，车载计算单元，用于人机交互的HMI Device和用于记录信息的黑匣子。其中传感器融合单元将所有传感器数据统一，完成时间戳的同步，在传输到后端计算单元进行处理。
   第三层是软件开发平台，这个模块在gitgub上开源。他可以看作是一个操作系统的底层框架加一个消息的分发机制。zai包括Map Engine，定位，感知，规划，控制等模块。Map Engine是车上的一个计算模块。当其他模块需要高精地图数据时，Map Engine负责从云端获取并处理。
   第四层是云端服务平台，包括高精地图，仿真服务，数据平台、安全、OTA和DuerOS。每一辆无人车可以将自己遇到的复杂路况上传到云端。当无人驾驶算法更新时，可以在仿真平台跑一下，检验是否能通过这些路况，通过了才进入真车测试，这样有效降低了生产成本和周期。

高精地图

   人们使用传统地图导航，导航地图会显示有几条路，预计需要多少时间等信息。但这些信息对于无人车来说远远不够，因为无人车缺乏人类驾驶员固有的视觉和逻辑能力，所以就需要高精度地图，来提供更多大量的驾驶辅助信息。其中包含道路定义、交叉路口、交通信号、车道规则以及用于汽车导航的其他元素。帮助无人车预先规划路线和行车操作，并辅助感知系统准确识别交通标识。

定位

   Apollo无人车定位使用GPS、惯性导航IMU和几何定位等一系列技术的融合。
   GPS更新频率低，IMU更新频率较高，利用卡尔曼滤波整合两类数据，可获得实时并准确的位置信息，但由于电离层、反射作用等因素的干扰，GPS定位是有误差的，它的精度只能达到米级，并且在封闭环境中无法使用，例如隧道。
   因此有了补充方案，摄像头捕捉道路上显著特征，与高精地图上的点计算车辆位置，但受照明条件约束。
   还有就是激光雷达产生点云进行“形状化描述”，利用粒子滤波将感测到的具体形状与地图比较，得到10cm精度内实时定位， 例如雨滴或灰尘。
   所以我们需要利用多种传感器的特点，整合出可靠并精准的定位。

传感

   目前主要的传感器有摄像头（Camera）、雷达（Radar）和激光雷达（Lidar）。
   摄像头可以对物体识别，主要用于检测车道线，识别红绿灯，人行道检测等。
   雷达利用电磁波探测目标方位，速度、高度等信息，准确度不高，善于判断速度。
   激光雷达通过发射激光探测目标位置，由于准确度非常高，是无人车主传感器。用来绘制高精地图、定位和避障。
